## Problem

The xFusionCorp Industries ML platform team's release checklist requires a five-metric evaluation report for every candidate model, plus a confusion-matrix image, published to the project's `reports/` directory. A draft `evaluate.py` exists for the pre-trained fraud-detection model, but the report it produces does not satisfy the checklist. The job is to correct the evaluator so the expected report lands in the right place.

1. The MLflow tracking server is already running on port `5000`, with an empty `fraud-detection-eval` experiment in place.
2. The project layout under `/root/code/fraud-detection/`:
   - `data/test.csv` — a 40-row held-out test set from an 80/20 stratified split.
   - `models/model.pkl` — a deterministic `RandomForestClassifier` pre-trained at lab startup. Do not retrain it.
   - `src/models/evaluate.py` — the evaluator draft (has bugs). Every concern other than the metrics report is correctly wired: confusion-matrix rendering, the MLflow run, the artifact logging.
   - `reports/` — where the metrics JSON and confusion-matrix image must land.
3. Open `src/models/evaluate.py` in VS Code, correct everything that prevents the end state from being reached, save, and run the script.
4. The end state must include:
   - A file at `/root/code/fraud-detection/reports/metrics.json` (absolute path, inside the project's reports directory).
   - That JSON contains exactly these five keys, each a numeric value: `accuracy`, `precision`, `recall`, `f1_score`, `auc_roc`.
   - A file at `/root/code/fraud-detection/reports/confusion_matrix.png`.
   - One MLflow run in the `fraud-detection-eval` experiment with the five metrics logged and both files attached as run artifacts.

## Solution

### Step 1: Read the buggy evaluator

```python
"""Evaluation script for the pre-trained fraud-detection model.

Loads the serialised model + held-out test set, produces a metrics
report and a confusion-matrix image, logs the results to MLflow.

Three deliberate problems prevent the report from satisfying the
lab's end state. Edit only the module-level constants and the
metrics computation inside `main()`; every other part of the
script — the confusion-matrix rendering, the MLflow wiring, the
persistence calls — is correct.
"""
import os
import json
import joblib
import pandas as pd
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt
import mlflow
from sklearn.metrics import (
    accuracy_score,
    precision_score,
    recall_score,
    f1_score,
    roc_auc_score,
    confusion_matrix,
)

MODEL_PATH = "/root/code/fraud-detection/models/model.pkl"
TEST_CSV = "/root/code/fraud-detection/data/test.csv"
REPORTS_DIR = "/root/code/fraud-detection/reports"
METRICS_JSON = "/tmp/metrics.json"
CONFUSION_PNG = os.path.join(REPORTS_DIR, "confusion_matrix.png")


def render_confusion_matrix(cm, path):
    fig, ax = plt.subplots(figsize=(4, 4))
    ax.imshow(cm, cmap="Blues")
    for i in range(cm.shape[0]):
        for j in range(cm.shape[1]):
            ax.text(j, i, str(cm[i, j]), ha="center", va="center", color="black")
    ax.set_xlabel("predicted")
    ax.set_ylabel("actual")
    ax.set_title("confusion matrix")
    fig.tight_layout()
    fig.savefig(path, dpi=100)
    plt.close(fig)


def main():
    model = joblib.load(MODEL_PATH)
    df = pd.read_csv(TEST_CSV)
    X = df.drop(columns=["is_fraud"])
    y = df["is_fraud"]

    preds = model.predict(X)
    proba = model.predict_proba(X)[:, 1]

    metrics = {
        "accuracy": round(accuracy_score(y, preds), 6),
        "f1": round(f1_score(y, preds), 6),
    }

    os.makedirs(REPORTS_DIR, exist_ok=True)
    os.makedirs(os.path.dirname(METRICS_JSON) or ".", exist_ok=True)
    with open(METRICS_JSON, "w") as f:
        json.dump(metrics, f, indent=2, sort_keys=True)

    cm = confusion_matrix(y, preds)
    render_confusion_matrix(cm, CONFUSION_PNG)

    mlflow.set_tracking_uri("http://localhost:5000")
    mlflow.set_experiment("fraud-detection-eval")
    with mlflow.start_run(run_name="evaluation"):
        for key, value in metrics.items():
            mlflow.log_metric(key, value)
        mlflow.log_artifact(CONFUSION_PNG)
        if os.path.exists(METRICS_JSON):
            mlflow.log_artifact(METRICS_JSON)

    print(f"metrics: {metrics}")
    print(f"metrics JSON: {METRICS_JSON}")
    print(f"confusion matrix: {CONFUSION_PNG}")


if __name__ == "__main__":
    main()
```

Three problems, exactly as the docstring warns:

1. `METRICS_JSON = "/tmp/metrics.json"` — wrong location entirely, outside the project tree.
2. The `metrics` dict only computes `accuracy` and `f1` (misnamed — should be `f1_score`). `precision`, `recall`, and `auc_roc` are missing, even though `precision_score`, `recall_score`, and `roc_auc_score` are already imported and `proba` is already computed for exactly this purpose.
3. (Consequence of #1) Once the path is fixed, the existing `mlflow.log_artifact(METRICS_JSON)` call — which was already correct — naturally attaches the file from its proper location.

### Step 2: Fix the constant and the metrics dict in VS Code



## Key Takeaways

### Accuracy alone hides exactly the failure mode that matters in fraud detection
This run's numbers tell a real story: precision is 0.73 but recall is 0.95 — the model is catching almost all actual fraud cases but at the cost of flagging some legitimate transactions too. A report that stopped at `accuracy: 0.8` would hide that trade-off completely. Precision, recall, and F1 together are what let a reviewer see *how* a model is wrong, not just *how often*.

### `predict_proba` exists for metrics that `predict` alone can't support
`roc_auc_score` needs continuous probability scores, not the final 0/1 class prediction — that's why the script computes `proba = model.predict_proba(X)[:, 1]` separately from `preds`. Any time a metric is missing from a report and the model's predict step only produces hard labels, check whether the underlying score needs `predict_proba` instead.
