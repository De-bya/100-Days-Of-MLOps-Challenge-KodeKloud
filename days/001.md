# Day 001: Python Virtual Environment for ML Projects

**Difficulty**: 🟢 Beginner 

---

## 🎯 Objective

The xFusionCorp Industries data science team needs a standardised Python environment for their new ML project. Set up a virtual environment with the required ML libraries on the `controlplane` host.

**Tasks:**
1. Create a Python virtual environment named `ml-env` under `/root/code/` using `python3 -m venv`
2. Activate the environment and install: `numpy`, `pandas`, `scikit-learn`, and `matplotlib`
3. Generate a `requirements.txt` file using `pip freeze` and save it at `/root/code/requirements.txt`

---

## 📝 Steps

### Step 1: Navigate to the project directory

```bash
cd /root/code
```

### Step 2: Create the virtual environment

```bash
python3 -m venv ml-env
```

This creates a `ml-env/` folder under `/root/code/` containing a self-contained Python environment.

### Step 3: Activate the virtual environment

```bash
source ml-env/bin/activate
```

Your terminal prompt will change to show `(ml-env)` — this confirms the environment is active.

### Step 4: Install the required ML packages

```bash
pip install numpy pandas scikit-learn matplotlib
```

Wait for all packages and their dependencies to finish installing.

### Step 5: Generate requirements.txt

```bash
pip freeze > /root/code/requirements.txt
```

### Step 6: Verify

```bash
cat /root/code/requirements.txt
```

You should see a full list of installed packages with pinned versions, including `numpy`, `pandas`, `scikit-learn`, and `matplotlib`.

---


## ✅ Verification & Troubleshooting

### Confirm packages are installed

```bash
python3 -c "import numpy, pandas, sklearn, matplotlib; print('All packages OK')"
```

### Common Issues

| Problem | Solution |
|--------|----------|
| `python3: command not found` | Install with `apt install python3` |
| `pip: command not found` | Install with `apt install python3-pip` |
| Permission denied on `/root/code` | Run with `sudo` or check directory permissions |
| Virtual env not activating | Ensure you use `source`, not just `./ml-env/bin/activate` |

---

## 💡 Key Takeaways

- **Virtual environments** isolate project dependencies — essential for reproducible ML projects
- `python3 -m venv` is the built-in, zero-dependency way to create environments
- `pip freeze` captures exact package versions, enabling reproducibility across machines
- Always commit `requirements.txt` to version control for team consistency

### Why Virtual Environments in ML?

ML projects often require specific library versions. Without isolation:
- Project A needs `numpy==1.21` but Project B needs `numpy==1.26`
- System-wide installs conflict and break

Virtual environments solve this completely.
