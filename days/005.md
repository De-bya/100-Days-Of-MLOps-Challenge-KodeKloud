# Day 005: Makefile for ML Project Automation

**Category**: ML Workflow Automation | **Difficulty**: 🟡 Intermediate

---

## Problem

The xFusionCorp Industries ML team uses a Makefile to orchestrate common tasks. A draft Makefile exists at `/root/code/fraud-detection/Makefile`, but `make all` does not complete successfully.

The corrected Makefile must declare six targets:
- `setup` - Creates a virtual environment at `mlops-venv/` and installs dependencies from `requirements.txt`
- `data` - Runs `python src/data/process_data.py`
- `train` - Runs `python src/models/train.py`
- `test` - Runs `pytest tests/`
- `clean` - Recursively removes every `__pycache__` directory, removes `.pytest_cache`, and clears `models/`
- `all` - Runs `setup`, `data`, `train`, and `test` in that order

All six targets must be declared as `.PHONY`.

---

## Solution

### Step 1: Inspect the broken Makefile

```bash
cat /root/code/fraud-detection/Makefile
```

### Step 2: Overwrite with the corrected version

```bash
cat > /root/code/fraud-detection/Makefile << 'EOF'
# fraud-detection Makefile
.PHONY: setup data train test clean all

setup:
	python3 -m venv mlops-venv && mlops-venv/bin/pip install -r requirements.txt

data:
	python src/data/process_data.py

train:
	python src/models/train.py

test:
	pytest tests/

clean:
	find . -type d -name __pycache__ -exec rm -rf {} +
	find . -type d -name .pytest_cache -exec rm -rf {} +
	rm -rf models/*

all: setup data train test
EOF
```

### Step 3: Run and verify

```bash
make all
```

## Key Takeaways

### What is a Makefile?
A Makefile is a simple automation file that defines **targets** (tasks) and the **commands** to run them. Originally designed for compiling C code, it is now widely used in MLOps to standardize how a team runs common tasks. Instead of remembering long commands, everyone just runs `make train` or `make setup` — same result every time, on every machine.

### Targets and `.PHONY`
A target is a named task in the Makefile. By default, Make checks if a file with that name exists — if it does, it skips the task. `.PHONY` tells Make: "this is always a command, never a file." Without it, if someone creates a file named `test`, running `make test` would do nothing. Always declare your targets as `.PHONY`.

### Tabs vs Spaces — the most infamous Makefile gotcha
Make is one of the few tools where tabs and spaces are not interchangeable. Every recipe line must start with a real tab character, not spaces. If you use spaces, Make throws `missing separator` and stops. When in doubt, use `cat -A Makefile` to inspect — tabs show as `^I`.

### Understanding `find . -type d -name __pycache__ -exec rm -rf {} +`

| Part | Meaning |
|---|---|
| `find .` | search from current directory |
| `-type d` | look for directories only |
| `-name __pycache__` | named `__pycache__` |
| `-exec rm -rf {} +` | delete each one found |

Simple `rm -rf __pycache__` only deletes the top-level folder. In an ML project with many submodules, `__pycache__` folders exist inside every package — the `find` version cleans all of them.

### The `all` target — the standard entry point
The `all` target runs the full pipeline from scratch: set up environment, process data, train model, run tests. Having a single `make all` command makes onboarding new teammates trivial and ensures CI/CD pipelines always run the same sequence.

### Why Makefiles in MLOps?
As ML projects grow, you accumulate many scripts and commands. Makefiles act as living documentation — they show exactly how the project is meant to be run and enforce consistency across local machines, CI runners, and production environments.
