## Key Takeaways

### 🏷️ Loading by alias — `models:/fraud-detector@champion`
The `models:/<name>@<alias>` URI syntax loads whichever version currently holds the alias, without hardcoding a version number. When the champion is promoted to a newer version, only the alias pointer changes — the serving code stays untouched.

### 🔌 `mlflow.pyfunc.load_model()` — the universal loader
`mlflow.pyfunc.load_model(uri)` works with any MLflow-supported flavour (sklearn, pytorch, tensorflow, custom pyfunc). It returns a `PyFuncModel` object with a `.predict()` method, making it the standard entry point for inference regardless of the underlying framework.

### 🧪 `mlflow.pyfunc.PythonModel` — custom preprocessing wrappers
Subclassing `PythonModel` and implementing `.predict()` lets you bundle arbitrary preprocessing logic with a model. Here, `ScaledPredictor` applies per-column mean/std scaling before calling the inner model — encapsulating the full inference pipeline in a single object rather than scattering preprocessing across calling code.

### 📦 Batch prediction pattern
The standard batch pattern is: load inputs → transform → predict → attach predictions as a new column → write to CSV. Keeping predictions in the same DataFrame as the inputs preserves row alignment and makes the output immediately auditable — each input row maps directly to its prediction.
